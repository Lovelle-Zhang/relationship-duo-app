# 关系显影剂 · 部署指南

## 第一步：注册 Supabase（存数据用）

1. 打开 https://supabase.com，点右上角 **Start your project**
2. 用 GitHub 账号或邮箱注册（免费）
3. 登录后点 **New project**，填写：
   - Name：relationship-duo（随意）
   - Database Password：设一个密码（记住）
   - Region：选 **Southeast Asia (Singapore)**（最快）
4. 点 **Create new project**，等约 1 分钟

5. 进入项目后，点左侧 **SQL Editor**，粘贴并运行以下 SQL：

```sql
CREATE TABLE pairs (
  id SERIAL PRIMARY KEY,
  code TEXT NOT NULL,
  role TEXT NOT NULL,
  scores JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(code, role)
);
```

6. 点左侧 **Project Settings → API**，记录：
   - **Project URL**（形如 https://xxxx.supabase.co）
   - **anon public** key（很长的一串）

---

## 第二步：注册 Vercel（部署网站用）

1. 打开 https://vercel.com，点 **Sign Up**
2. 选 **Continue with GitHub**（需要先有 GitHub 账号）
   - 没有 GitHub？先去 https://github.com 注册，免费

---

## 第三步：把代码上传到 GitHub

1. 打开 https://github.com，点右上角 **+** → **New repository**
2. Repository name：relationship-duo
3. 选 **Private**，点 **Create repository**
4. 打开你电脑上的 `relationship-duo-app` 文件夹
5. 把文件夹里所有文件拖进 GitHub 页面上传
   （或用 GitHub Desktop 客户端上传，更简单）

---

## 第四步：在 Vercel 部署

1. 打开 https://vercel.com/dashboard
2. 点 **Add New → Project**
3. 选择你刚创建的 **relationship-duo** 仓库，点 **Import**
4. 什么都不用改，直接点 **Deploy**
5. 等约 1 分钟，部署完成

---

## 第五步：配置环境变量（最关键）

1. 在 Vercel 项目页，点 **Settings → Environment Variables**
2. 添加两个变量：

| Name | Value |
|------|-------|
| SUPABASE_URL | 你的 Project URL |
| SUPABASE_ANON_KEY | 你的 anon public key |

3. 点 **Save**
4. 点 **Deployments → 最新一条 → Redeploy**（重新部署让变量生效）

---

## 第六步：完成！

Vercel 会给你一个免费域名，形如：
**https://relationship-duo-xxxx.vercel.app**

把这个链接发给另一半，就可以跨设备配对了 🎉

---

## 常见问题

**Q：全程免费吗？**
A：是的。Supabase 免费版支持 50MB 数据库，Vercel 免费版支持每月 100GB 流量，完全够用。

**Q：数据安全吗？**
A：数据存在 Supabase 的服务器上，只有知道邀请码的人才能读取对应数据。

**Q：邀请码多久失效？**
A：数据永久保存，不会失效。如需清理可以登录 Supabase 手动删除。
