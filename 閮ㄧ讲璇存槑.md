# 微信支付部署说明

## 1. 文件结构

将以下文件放到你的 GitHub 仓库：

```
relationship-duo-app/
├── index.html              (已有，需要修改unlockInsight函数)
└── api/
    ├── create-order.js     (创建订单)
    └── notify.js           (支付回调)
```

## 2. 修改 index.html

找到 `unlockInsight()` 函数，替换为：

```javascript

      headers: {'Content-Type': 'application/json'},

    const data = await res.json();
    
    if (data.code_url) {

    alert('支付系统错误：' + error.message);
```

## 3. 在 index.html 中引入二维码库

在 `</head>` 前添加：

```html
<script src="https://cdn.jsdelivr.net/npm/qrcodejs@1.0.0/qrcode.min.js"></script>
```

## 4. 部署到 Vercel

1. 将 `api/` 文件夹放到 GitHub 仓库根目录
2. 确保项目结构如上所示
3. Vercel 会自动识别 `api/` 目录并部署为 Serverless Functions
4. 等待 Vercel 部署完成

## 5. 测试

1. 访问网站，完成测试
2. 点击「立即解锁」
3. 应该弹出微信支付二维码
4. 用微信扫码支付 ¥29
5. 支付成功后内容自动解锁

## 6. 注意事项

- 确保回调地址配置正确：`https://relationship-duo-app-nu.vercel.app/api/notify`
- 在微信商户平台配置此回调地址白名单
- 测试时可以先用 0.01 元测试

## 7. 问题排查

如果支付失败，检查：
1. Vercel 的 Functions 日志
2. 微信商户平台的交易记录
3. 浏览器 Console 的错误信息
